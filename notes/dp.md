# Dynamic Programming (DP)

## Key Concepts
- Break problem into overlapping subproblems.
- Store computed states to avoid repeated work.
- Two styles: top-down (memoization) and bottom-up (tabulation).

## Time Complexity
- Depends on number of states and transitions.
- Typical formula: **O(states × transitions)**

## Common Patterns
- 1D DP (Fibonacci, climbing stairs)
- 2D DP (grid paths, knapsack)
- Include/exclude decisions
- Longest subsequence style problems
